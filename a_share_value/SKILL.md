# a_share_value - A股价值抄底选股策略

一套"三阶筛选+量化评分"的A股价值抄底体系，核心在于区分"价格低估"与"价值陷阱"。

> **skill名称**: `a_share_value` (A股价值选股)
> **策略类型**: 价值投资 / 逆向投资 / 长期持有
> **投资期限**: 1年以上

---

## 概述

本策略构建了一套完整的三阶筛选体系，通过初阶估值过滤、精阶质量验证、高阶成长判断，最终形成80分以上的核心池标的，配合分批建仓与动态再平衡机制，实现长期复利收益。

### 与趋势策略的区别

| 维度 | `a_share_value` (本策略) | `a_share_trend` (趋势策略) |
|------|-------------------------|---------------------------|
| **核心理念** | 逆向投资，逢低买入 | 趋势跟踪，追涨杀跌 |
| **选股标准** | 低PE/PB、高股息、高ROE | 创新高、放量、均线多头排列 |
| **持仓周期** | 1年以上 | 2-4周 |
| **风险特征** | 低波动、高安全边际 | 中高波动、动量延续 |
| **适用市场** | 震荡市、熊市后期 | 牛市、强势市场 |

## 核心框架

### 三阶筛选体系

```
┌─────────────────────────────────────────────────────────────┐
│  初阶筛选：估值安全边际                                        │
│  - PE < 15倍 / PB < 1.5倍                                    │
│  - 股息率 > 3%                                               │
│  - 排除ST/*ST、退市风险                                       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  精阶筛选：盈利能力与财务健康度                                 │
│  - ROE连续3年 > 12%                                          │
│  - 经营现金流/净利润 > 60%                                    │
│  - 资产负债率 < 60%                                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  高阶筛选：成长性验证与错杀时机判断                             │
│  - 近三年净利润CAGR > 5%                                      │
│  - 识别错杀信号（PE折价>30%、高股息+低估值）                    │
└─────────────────────────────────────────────────────────────┘
```

### 多因子评分模型

| 维度 | 权重 | 核心指标 | 满分标准 |
|------|------|----------|----------|
| **估值安全** | 40% | PE历史分位、PB、股息率 | PE<10%分位+PB<1+股息率>5% |
| **盈利质量** | 25% | ROE、毛利率 | ROE>20%+毛利率>50% |
| **财务健康** | 20% | 现金流、负债率 | 现金流/利润>100%+负债率<50% |
| **成长潜力** | 15% | 净利润CAGR、营收CAGR | 利润CAGR>20%+营收CAGR>15% |

### 分级标准

| 等级 | 分数区间 | 应用建议 | 仓位上限 |
|------|----------|----------|----------|
| 🔴 核心池 | ≥80分 | 优先配置，深度研究后可集中持仓 | 15% |
| 🟡 观察池 | 60-80分 | 持续跟踪，待瑕疵改善后介入 | 8% |
| 🔵 备选池 | 40-60分 | 关注特定催化因素 | 3% |
| ⚪ 排除 | <40分 | 不符合价值抄底标准 | 0% |

## 快速开始

### 1. 执行完整选股流程

```bash
cd /Users/lijian/.agents/skills/a_share_value

# 标准选股（保守型）
python3 run.py

# 放宽条件（进攻型）
python3 run.py -p 20 -b 2.0 -d 2.0

# 严格条件（极致安全边际）
python3 run.py -p 10 -b 1.0 -d 4.0
```

### 2. 分析指定股票

```bash
python3 run.py --code 000001.SZ
```

### 3. 代码中使用

```python
import tushare as ts
from value_selector import ValueSelector

pro = ts.pro_api()
selector = ValueSelector(pro)

# 执行选股
result = selector.select_stocks(
    max_pe=15,              # 最大PE
    max_pb=1.5,             # 最大PB
    min_dividend_yield=3.0, # 最小股息率
    top_n=30                # 选出数量
)

# 查看结果
print(result['core_pool'])      # 核心池
print(result['watch_pool'])     # 观察池
```

## 模块说明

### valuation_filter.py - 估值筛选模块

**核心类**: `ValuationFilter`

**主要方法**:

| 方法 | 说明 |
|------|------|
| `preliminary_filter(max_pe, max_pb, min_dividend_yield)` | 初阶筛选 |
| `get_pe_history_percentile(ts_code)` | 获取PE历史分位数 |
| `check_dividend_sustainability(ts_code)` | 检查股息可持续性 |
| `get_industry_pe_median(industry)` | 获取行业PE中位数 |

### quality_filter.py - 质量筛选模块

**核心类**: `QualityFilter`

**主要方法**:

| 方法 | 说明 |
|------|------|
| `analyze_roe(ts_code)` | ROE质量分析 |
| `analyze_gross_margin(ts_code)` | 毛利率分析 |
| `analyze_cashflow(ts_code)` | 现金流分析 |
| `check_financial_risk(ts_code)` | 财务风险排雷 |
| `comprehensive_quality_check(ts_code)` | 综合质量检查 |

### growth_analyzer.py - 成长性分析模块

**核心类**: `GrowthAnalyzer`

**主要方法**:

| 方法 | 说明 |
|------|------|
| `calculate_growth_rates(ts_code)` | 计算增长率CAGR |
| `analyze_growth_drivers(ts_code)` | 分析增长驱动因素 |
| `detect_mispricing(ts_code)` | 检测错杀信号 |
| `comprehensive_growth_analysis(ts_code)` | 综合成长性分析 |

### value_scorer.py - 多因子评分模块

**核心类**: `ValueScorer`

**主要方法**:

| 方法 | 说明 |
|------|------|
| `calculate_total_score(ts_code)` | 计算综合评分 |
| `get_pool_recommendation(score_result)` | 获取仓位建议 |

### value_selector.py - 选股主模块

**核心类**: `ValueSelector`

**主要方法**:

| 方法 | 说明 |
|------|------|
| `select_stocks(max_pe, max_pb, min_dividend_yield, top_n)` | 执行完整选股流程 |
| `step1_valuation_filter()` | 初阶筛选 |
| `step2_quality_filter()` | 精阶筛选 |
| `step3_growth_filter()` | 高阶筛选 |
| `step4_comprehensive_scoring()` | 综合评分 |

## 选股标准详解

### 保守型价值组合

适合风险厌恶型投资者，强调极致安全边际：

```python
selector.select_stocks(
    max_pe=15,              # PE < 15倍
    max_pb=1.5,             # PB < 1.5倍
    min_dividend_yield=4.0  # 股息率 > 4%
)
```

### 进攻型价值组合

适合风险承受能力较高、追求估值修复弹性的投资者：

```python
selector.select_stocks(
    max_pe=20,              # PE < 20倍
    max_pb=2.0,             # PB < 2.0倍
    min_dividend_yield=3.0  # 股息率 > 3%
)
```

## 典型错杀场景识别

### 行业性错杀

| 场景类型 | 典型案例 | 识别要点 | 布局时机 |
|----------|----------|----------|----------|
| 政策冲击后的优质龙头 | 医药集采后的创新药企 | 政策影响的差异化 | 政策底出现 |
| 技术路线争议期 | 新能源转型期的传统车企 | 传统业务现金流 | 转型进展验证 |
| 周期底部 | 基建、有色金属 | 产能利用率拐点 | 领先指标改善 |

### 个体性错杀

| 场景类型 | 分析框架 | 验证要点 |
|----------|----------|----------|
| 短期业绩miss但长期逻辑 intact | 区分"一次性冲击"vs"趋势性恶化" | 核心竞争力是否受损 |
| 大股东变更或治理改善预期 | 评估治理改善的可信度 | 新管理层履历、改革措施 |
| 机构流动性冲击后的修复 | 分析筹码结构变化 | 股东户数减少、调研增加 |

## 建仓与仓位管理

### 分批建仓机制

| 阶段 | 触发条件 | 仓位比例 | 核心验证 |
|------|----------|----------|----------|
| 首次建仓 | 评分进入核心池（≥80分） | 30%目标仓位 | 估值安全边际确认 |
| 加仓触发 | 股价下跌10%且基本面无变化 | 60%目标仓位 | 季报验证 |
| 满仓条件 | 估值处于历史10%分位且股息率>5% | 100%目标仓位 | 长期风险收益比最优 |

### 止损与止盈纪律

| 类型 | 条件 | 执行方式 |
|------|------|----------|
| 硬性止损 | 单标亏损20% 且 基本面恶化确认 | 果断止损 |
| 动态止盈-第一阶段 | 估值修复至历史均值（PE分位50%） | 减仓50% |
| 动态止盈-第二阶段 | 估值处于历史高估区间（PE分位>80%） | 完全退出 |

## 注意事项

### 频率限制说明 ⚠️

Tushare API 有频率限制（免费用户通常每分钟 500 次）。策略已内置频率控制机制：

1. **自动延时**：程序会自动控制 API 调用速度，避免触发限制
2. **自动重试**：如果遇到频率限制，会自动等待 10 秒后重试
3. **缓存机制**：基础数据（如股票列表、交易日历）会缓存 10 分钟
4. **批量处理**：评分时会批量处理，避免过快调用

**提示**：
- 选股过程可能需要 1-5 分钟，请耐心等待
- 若频繁遇到限制，可适当调低 `top_n` 参数减少评分数量
- 建议避免在交易时段（9:30-15:00）进行大量数据获取

### 数据与策略

1. **数据更新**：Tushare数据有1-2天延迟，不适合日内交易决策

2. **权限要求**：部分接口需要Tushare积分：
   - 财务指标数据
   - 分红数据
   - 机构持仓数据

3. **策略适配**：
   - 本策略在**熊市后期**和**震荡市**表现最佳
   - **牛市初期**可能错过部分成长型标的
   - **高通胀环境**需提高股息率门槛

4. **风险控制**：
   - 单一标的上限不超过15%
   - 单一行业上限不超过30%
   - 建立"核心-卫星"组合结构
